Movement Analysis

This project analyzes home ranges and detected clusterss of the spotted Nutcracker (Nucifraga caryocatactes) using GPS and sensor data from 115 tagged individuals.

Research Questions

This will be our research questions for the project:

  • Research Question 1

    What are the home ranges of the birds, and to what extent do they overlap?

  • Research Question 2

    How do the home ranges vary across different seasons?

  • Research Question 3

    How effective is DBSCAN in identifying bird clustering patterns?

  • Research Question 4

    How effective can an R Shiny application be in detecting and visualising the patterns addressed in Research Questions 1–3?

    More curious (Code & More Information)? Click here to view it.

    The main preprocessing is described in the file: Preprocessing_RShiny.qmd

    Overview of Preprocessing Steps
    • Data cleaning: removed missing coordinates and known faulty entries.
    • Date conversion: standardised timestamps for temporal filtering.
    • Spatial conversion: transformed data into spatial features (using sf).
    • Season tagging: assigned each point to a seasonal category (Breeding, Harvesting, Winter).
    • Track generation: created bird movement lines per ID and month.
    • Clustering (DBScan): applied density-based clustering for each bird across seasons and overall.
    • Export: saved processed data as .geojson and .csv for direct app integration.
    Additional Processing
    • Visual inspection of clusters to verify correctness (with tmap).
    • Convex hull generation: created spatial polygons around clusters.
    • Overlap matrix: calculated spatial overlaps between birds to study shared areas.
    • Helper functions: provided tools to query overlapping bird IDs.

  • RShiny Development

    R (done by: Claude)

    We developed the entire interactive dashboard using R Shiny. It allows users to explore, filter, and compare bird tracking data through a dynamic and user-friendly web interface.

    Show R Shiny Overview and Logic

    Overview

    This section provides an overview of the interactive dashboard we built using R Shiny. The main goal was to create a simple and intuitive interface to explore bird tracking data, compare individuals, and analyse movement patterns.

    Users can easily navigate through individual birds using Next/Previous buttons, apply seasonal filters, and switch between various statistical views. The interactive map updates dynamically based on user input.

    A help (?) button is placed in the top right corner to offer clear guidance and explain each dashboard feature. This ensures that new users can easily understand the app without prior training.

    How the Code Works

    The app is built around the ui and server components of R Shiny. Data is loaded and pre-processed from GeoJSON and CSV files.

    - reactive() and observeEvent() are used to update visual elements when the user selects a different bird or season.
    - The mbp variable stores the 95% Minimum Bounding Polygon (MCP), which shows the home range of a bird based on spatial data. It is calculated with st_convex_hull() and filtered by distance quantiles.
    - All maps are rendered using leaflet, with overlays for Tracks, Points, Clusters, and Home Ranges.
    - Multiple plots are created using ggplot2 and updated dynamically with seasonal or individual bird data.

    For easier access, we have published our code on our GitHub Project. You can find it here: GitHub – GEO880 Project Repository
    The main preprocessing is described in the file: RShiny_Map.r

  • Website Creation

    HTML/CSS/JS (Bootstrap 5 Framework / done by: Claude)

    The project website was built using Bootstrap 5. It provides a clean and responsive layout, styled components, and an interactive timeline to present all project stages.

    Show Website Overview and Logic

    Overview

    We created the website using Bootstrap 5, a modern CSS framework that enables mobile-first, responsive web design. The grid system was used to align content and components in a structured and flexible way.

    All sections are wrapped in a vertical timeline layout, with icons, expandable cards, and call-to-action buttons. It’s designed to work well across different devices and screen sizes.

    How the Code Works

    The layout uses the .container, .row, and .col-* classes to ensure proper spacing and alignment.

    Each project phase is a .timeline-item with custom icons and collapsible content using data-bs-toggle="collapse".

    Additional Bootstrap classes like .btn, .card, and .text-muted were used to ensure a clean and consistent design.
